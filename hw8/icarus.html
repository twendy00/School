How to Publish a Website to the Icarus Server

Upload to Icarus

  • First, make sure that you have a FTP client. If not, there are several available that you can download such as WinSCP, FileZilla, etc. Personally, I use Cyberduck.
  • Open your FTP client, and open a new connection. Change the first drop down box to SFTP (SSH File Transfer protocol).
  • In the Server text box, enter the following: icarus.cs.weber.edu. The port should remain as 22.
  • Your username will be your first and last initials followed by the last 5 digits of your W-number.
  • Your password is your first name (first letter is capitalized) followed by cs!.
  • Once connected, create a new folder or select an existing folder to upload a file.
  • To upload a single file, select Action (the gear symbol) and then Upload. Select the file you'd like to upload and click Upload.
  • To upload multiple files or an entire folder, you can drag and drop or copy and paste the items into your FTP client.

Compose a URL

When you compose a URL, it will consist of the following components (in this order):

Protocol, server/host name, forward slash, tilde, username for your account, forward slash, the path to the file, forward slash, file name, forward slash.

So putting all of this together in an example:

  • Your protocol will be: http://
  • The server/host name is: icarus.cs.weber.edu
  • In this example, the user name is wt65542.
  • In this example, the path and file name will be cs2350/la2files/distractions.html/

http://icarus.cs.weber.edu/~wt65542/cs2350/la2files/distractions.html

However, if you do not want the file name to show in the URL link, you can name the file as index.html. This is typically done for a default or home page. For the above example, if the file distractions.html was actually called index.html, the URL link would now be:

http://icarus.cs.weber.edu/~wt65542/cs2350/la2files/

More examples can be found below.

Example 1

http://icarus.cs.weber.edu/~wt65542/meals.html

In this example, the file is saved directly into the public_html folder. It is not saved in any other folder, so the URL does not need to reference a path or any other folder. After your username, you can reference the file name directly.

Example 2

http://icarus.cs.weber.edu/~wt65542/cs2350/pa2/wise.html

In this example, the file is saved in 2 folders beyond public_html. It is saved in the folder CS2350 and pa2. The URL needs to reference both of these folders, or the path, first before it can reference the file.

Example 3

http://icarus.cs.weber.edu/~wt65542/cs2350/la2files/

In this example, the file is saved as an index.html file, so the file name will not need to be included in the URL. However, since this file is still 2 folders deep into public_html, the path will still need to be specified.
